# Welcome to Test Intelligence Mesh

Test Intelligence Mesh is a layered learning system that connects AI agents, prompting skills, and quality engineering practices. It's for testers and quality engineers who want to use AI tools practically — not just experimentally.

## Why "mesh"?

A mesh is strong because of its connections, not despite them. Each fiber alone is fragile. Together, they hold weight.

That's how we think about AI in testing. Individual prompts are useful. Connected workflows — where agents hand off context to each other, where prompts build on domain knowledge, where humans stay in the loop — are powerful.

We're building this one fiber at a time. The mesh grows as we learn what works.

## Who this is for

If you've heard about AI tools like GitHub Copilot or Claude Code and wondered what they can concretely do for testing work — this is for you.

If you've tried a few prompts and want a structured path to doing more — this is for you.

If you want to understand what "AI for quality engineering" means without the hype — definitely for you.

This project doesn't assume you're a developer. It doesn't assume you have a computer science background. It assumes you care about quality and you're curious.

## What you'll find here

**Agents** — six practical agents organized by learning level (L1–L3). Each one has a concrete description of what it does, why you'd want it, and step-by-step instructions for both GitHub Copilot and Claude Code.

**Blog** — articles on using AI in quality engineering, written from practice. When we learn something that changes how we work, we write it down here.

## How to get started

Go to the [Agents](/agents.html) page and start with the L1 agents. Try the **Test Scenario Explorer** on a real user story from your current project — not a toy example. See what comes back. It takes about 90 seconds.

Notice what's good. Notice what's wrong. Notice how you instinctively want to correct it. That noticing is the first skill the mesh teaches.

Then read [Skills, Combos, Chains](/post.html?id=02-layered-learning) to understand how the moves build on each other — and why you decide when to progress.

One fiber at a time.
